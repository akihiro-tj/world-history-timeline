# 世界史タイムライン 設計書

## 概要

世界史の「誰が・いつ・どこで」を直感的に掴むための年表 Web アプリ。共通の時間軸上に地域ごとのレーンを並べ、各地の統治者・人物・事件を対照表示することで、「その頃、他の地域では何が起きていたか」という同時代性を一目で把握できるようにする。大学受験や歴史能力検定などの試験対策から、教養としての学び直しまで、世界史学習全般の支援を目的とする。

例: イングランドでエドワード1世が統治していた頃、フランスではフィリップ4世が教皇と争い、東方ではマルコ・ポーロが元に滞在していた——この並行関係が視線の横移動だけでわかる画面を目指す。

## MVP スコープ

- 公開の静的 Web アプリ（GitHub Pages）。スマホ・PC の両方で利用できる
- 時代範囲は古代〜現代までを最初から扱い、データは薄く始めて後から密にする
- 縦軸タイムライン（上→下に時間が流れる）× 地域単位の固定レーン（左右に並ぶ列）
- エントリ種別は統治者（ruler）・人物（person）・事件（event）の3種
- 連続ズーム＋縦スクロールによるナビゲーション
- エントリのタップ／クリックで詳細パネルを表示（同時代の他地域エントリへのジャンプ動線を含む）
- 名前検索と年代ジャンプ
- データはリポジトリ内の JSON を手作業でキュレーションする

### MVP に含めないもの（将来拡張）

- 王朝・国家の存続期間を示す背景帯レイヤー
- ズーム位置・選択状態の URL 共有
- 暗記チェック（クイズ）モード
- 読み仮名・別名による検索
- 横軸レイアウトへの切替
- 学習記録

## アーキテクチャ

```
world-history-timeline/
├── public/
│   └── data/
│       ├── config.json  # 表示範囲などのグローバル設定
│       ├── regions.json # レーン（地域）定義
│       └── entries.json # 年表エントリ
├── src/
│   ├── domain/          # React 非依存の純粋ロジック
│   │   ├── scale.ts     # 年 → Y 座標の線形変換
│   │   ├── packing.ts   # レーン内サブカラム配置
│   │   ├── ticks.ts     # 年目盛の間隔選択
│   │   ├── visibility.ts# importance によるフィルタ
│   │   └── query.ts     # 検索クエリ解釈（年代 / 名前判定）
│   ├── components/      # タイムライン・詳細パネル・検索などの UI
│   └── ...
├── scripts/
│   └── validate-data.ts # zod によるデータ検証（ビルド前・CI で実行）
└── .github/workflows/   # CI（typecheck → test → build）と Pages デプロイ
```

- サーバー・API は持たない。JSON は `public/data/` に置き、アプリ起動時に並列 fetch する。JS バンドルとデータを分離することで、データが数千件に増えてもバンドルサイズと JS パース時間に影響しない。ブラウザキャッシュも JS と別々に効く
- zod スキーマはアプリ本体と検証スクリプトで共用し、実行時も fetch 結果を parse してから使う（型の一次情報をスキーマに一元化する）
- 座標計算・配置計算などのロジックは `src/domain/` に純粋関数として分離し、単体テストの主対象とする
- データ検証はビルドの前段で必ず実行し、不正データは CI で落とす（後述）

## データモデル

### config.json

```jsonc
{
  "minYear": -3000,
  "maxYear": 2100
}
```

タイムラインの表示範囲。ズームレンジのクランプ計算にも使う。

### regions.json

```jsonc
[
  { "id": "west-europe", "name": "西欧", "order": 1, "color": "#4a90d9" },
  { "id": "east-europe", "name": "東欧・ロシア", "order": 2, "color": "#7b6bd9" }
]
```

初期レーン案（8レーン、`order` は左からの表示順）:

1. 西欧
2. 東欧・ロシア（ビザンツを含む）
3. 中東・北アフリカ
4. 中央ユーラシア（遊牧勢力）
5. 南アジア
6. 東アジア
7. 日本
8. アメリカ大陸

レーンはデータ定義であり、構成の変更はコード変更を伴わない。

### entries.json

```jsonc
[
  {
    "id": "edward-1",
    "type": "ruler",
    "region": "west-europe",
    "group": "england",
    "groupName": "イングランド",
    "title": "エドワード1世",
    "start": 1272,
    "end": 1307,
    "importance": 2,
    "description": "ウェールズを征服し、模範議会を招集。スコットランド遠征中に病没。"
  }
]
```

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | string | 一意な識別子（英小文字ケバブケース） |
| `type` | `"ruler" \| "person" \| "event"` | 種別。色・形の描き分けと検索フィルタに使用。配置ロジックは共通 |
| `region` | string | 所属レーンの ID。`regions.json` に存在しなければ検証エラー |
| `group` | string（任意） | 国・勢力の識別子。レーン内サブカラムの安定配置に使用 |
| `groupName` | string（`group` があれば必須） | 国・勢力の表示名 |
| `title` | string | 表示名 |
| `start` | number | 開始年。整数のみ（月日は持たない）。紀元前は負数（前300年 → `-300`） |
| `end` | number（`event` のみ任意） | 終了年。`ruler` / `person` では必須。`event` で省略した場合は点イベントとして扱う。`start <= end` を検証 |
| `importance` | 1〜3 | 表示階層。1 = 最重要（ズームアウト時も表示）、3 = 詳細 |
| `description` | string | 一言解説。詳細パネルに表示 |

紀元前の年は負数で保持し、表示時のみ「前300年」の形式に変換する。内部計算はすべて整数の線形軸で行う。

### データ検証（scripts/validate-data.ts）

zod スキーマで以下を検証し、違反があればビルドを失敗させる:

- 各フィールドの型・必須性・`importance` の範囲
- `id` の一意性
- `region` が `regions.json` に存在すること
- `ruler` / `person` に `end` が存在すること
- `end` が存在する場合 `start <= end`
- `start` / `end` が `config.json` の表示範囲内であること

手作成データの品質はこの検証が主防衛線となる。実行時のデータ起因エラーは原理的に発生しない構成とする。

## 描画・レイアウト

### 画面の基本構造

```
┌─────────┬──────────────────────────────────────────┐
│ (固定)   │  西欧        │ 東欧    │ 中東   │ 東アジア │ ← レーンヘッダー（上部 sticky）
├─────────┼──────────────┼─────────┼────────┼─────────┤
│ 1250 ─  │ ┌イングランド┐│         │┌ﾏﾑﾙｰｸ┐│ ┌ 元 ┐  │
│         │ │ヘンリ3世   ││        ││ …   ││ │ﾌﾋﾞﾗｲ│  │
│ 1275 ─  │ ├────────────┤│        │└──────┘│ │     │  │
│         │ │エドワード1世││ ◆ﾓﾝｺﾞﾙ │        │ ├─────┤  │
│ 1300 ─  │ │            ││  西征⋯ │        │ │成宗 │  │
│         │ └────────────┘│        │        │ └─────┘  │
└─────────┴──────────────┴─────────┴────────┴─────────┘
  ↑ 年軸（左端 sticky）
```

- スクロールコンテナ内に「表示範囲の総年数 × pxPerYear」の高さを持つ SVG を置き、ブラウザのネイティブ縦スクロールをそのまま使う
- 可視範囲＋前後マージンに交差するエントリのみ描画する（ビューポートカリング）
- `end` を持つエントリ（統治者の在位、人物の活動期間、百年戦争のような期間を持つ事件）は縦長の角丸バー、点イベント（`event` で `end` なし）は ◆ マーク＋ラベルで描画
- 種別は色相・形で描き分け、地域色はレーン背景の薄い色として使う
- 同じ高さ＝同じ年代。視線の横移動だけで同時代の各地域が対照できる

### レーン内の配置アルゴリズム（group パッキング）

「イングランド王は常に同じ縦ラインに並ぶ」を、幅を無駄にせず実現する:

1. レーン内のエントリを `group` でまとめる（`group` なしのエントリは自身単独のグループとして扱う）
2. 各グループの時間スパン = 所属エントリの最小 `start` 〜 最大 `end`
3. グループのスパン同士を、`start` 昇順の貪欲法でサブカラムに詰める（時間が重ならないグループは同じカラムを共有する）
4. グループ内では各エントリを1本のカラムに時系列で積む。共同統治などで期間が重なる場合のみ、同様の貪欲法でグループ内をさらに分割する

これにより、時代が重ならない勢力（例: ローマ帝国と後世のイングランド）はカラムを共有でき、全時代表示でもレーン幅が爆発しない。この計算は純粋関数として実装する。

### ラベル描画

- バーの描画高がラベルの最小表示高を下回る場合、ラベルを省略しバーのみ描画する
- バーが長い場合、ラベルは可視領域内に留まるよう追従させる（スクロールしても名前が読める）

## ズーム・ナビゲーション

- ズーム状態は `pxPerYear`（年あたりピクセル数）の1変数に集約する
- 操作方法: Ctrl/⌘＋マウスホイール、ピンチ、＋/−ボタン（ZoomControls）
- ズーム時はカーソル／ピンチ中心にある年が画面上で動かないようにスクロール位置を補正する
- ズームレンジは「全時代が2〜3画面に収まる」〜「1世紀が1画面」の範囲でクランプする（`config.json` の表示範囲から算出）
- 年目盛は密度に応じて 500年 → 100年 → 50年 → 10年 と自動切替する
- importance フィルタ: `pxPerYear` のしきい値で表示階層を切り替える（引き = importance 1 のみ → 寄るにつれ 2、3 が出現）。しきい値は調整可能な定数とする
- ZoomControls に「全体表示」ボタンを置き、ワンタップで全時代ビューに戻れるようにする

├── TimelineView         # スクロール容器・ズームイベント処理

### 詳細パネル

エントリのタップ／クリックで開く。表示内容:

- 名前・期間（紀元前表記対応）・種別・所属（`groupName`）・解説文（`description`）
- 「同時代」リスト: 選択エントリと期間が重なる他地域のエントリ（点イベントはその年で判定）。importance 1〜2 に限定し、地域順に最大10件表示。項目をタップするとそのエントリへスクロール＆選択が移る

「その頃、他の地域では」を明示的な動線として持つ。区間交差判定（`a.start <= b.end && b.start <= a.end`）の全件走査で実装する（数百件規模では十分高速）。

### 検索・年代ジャンプ

入力欄1つで兼用し、`domain/query.ts` の `parseQuery` で解釈する:

- `1300` のような数字 → 西暦年ジャンプ
- `前300` の形式 → 紀元前年ジャンプ
- それ以外の文字列 → `title` の部分一致で候補をリスト表示。選択でスクロール＆該当エントリをハイライト選択

全データはメモリ上にあるため線形フィルタで実装する。

### アクセシビリティ

- エントリ要素はフォーカス可能とし、名前と期間を aria-label で提供する
- 詳細パネルの開閉はキーボードでも操作できるようにする

## エラー処理

- 主防衛線はビルド時のデータ検証（前述）。データ内容に起因する実行時エラーは発生させない
- データ取得が完了するまではローディング表示を出し、fetch の失敗（オフライン等）時は再試行導線つきのエラー表示に切り替える
- 最上位に React Error Boundary を置き、描画時の想定外の例外に対しては再読み込み導線つきのフォールバック画面を表示する

## CI/CD

- PR ごと: データ検証 → typecheck → lint → test → build を GitHub Actions で実行
- main マージ: 上記に加えて GitHub Pages へ自動デプロイ
- Vite の `base` はリポジトリ名に合わせて設定する

## 初期データ方針

- 8レーン × 全時代で 150〜300 エントリ（importance 1〜2 中心）から開始し、学習の進みに合わせて追記する
- 記述は日本語。解説文は教科書レベルの周知の事実に基づき、既存書籍からの転載はしない
